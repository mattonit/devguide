Developer Guide by @mattonit

The collection of good practices for HTML and CSS to create proper and valid websites

Key Rules

  • One of my favorite rules in programming is: every single line of code should appear to be written by a single person.

  • Don't repeat your own code. Your code should be re-useable.

HTML5 Doctype

  • <!DOCTYPE html> must be the very first tag in your HTML document before <html> tag
  • HTML5 Dcotype declaration isn't case sensitive
  • Always add <!DOCTYPE html> to your HTML document. This will let the browser know what kind of content to expect
<!DOCTYPE html>
<html>
    <head>
    </head>
</html>

Character encoding

You should always declare charset in your HTML document to make sure that your content will be rendered correctly. Remember that charset declaration must be added to the <head> section.

<head>
    <meta charset="UTF-8">
</head>

For HTML4 you should use text/css and text/javascript when including CSS and JavaScript files. In HTML5 there's no need to specify the type

<!-- External CSS -->
<link rel="stylesheet" src="devguide.css">

<!-- In-document CSS -->
<style>
    /* ... */
</style>

<!-- JavaScript -->
<script src="code-guide.js"></script>

Internet Explorer Version

In some cases, you might need to choose what version of Internet Explorer the page should be rendered as.

It's recommended to put X-UA-Compatible tag as high as possible in your document head


<meta http-equiv="x-ua-compatible" content="IE=8">

<meta http-equiv="x-ua-compatible" content="IE=EmulateIE7">

<meta http-equiv="x-ua-compatible" content="IE=edge">

Always Validate

To make sure that your code use HTML and CSS standards you should always validate your code. That will help you to create websites that would be rendered properly.

Validators will also help you to debug your code. Using them you get the list of things that you should fix.

  • HTML Validator
  • CSS Validator
  • Validator for HTML5 documents

Classes names

  • Class names should be simply and self-describing. They should be understandable clearly define the destination of the class.

  • You should also avoid shortcuts that would be hard to understand. While .btn is a good class name for button, .b doesn't mean anything

  • Keep classes names as short as possible

/* Good example */
.no-padding {
    padding: 0;
}

/* Bad example */
.np {
    padding: 0;
}

Reset CSS removes all browser built-in styles. That will make all of the HTML elements completely unstyled Normalize CSS makes built-in browser styles look consistent across browsers

Don't use too many webfonts

You shouldn't use too many webfonts on your website. When you load too many webfonts your page will load longer.

Also, when you use a lot of different fonts, your website may be less pretty and confuse your users.

It's recommended to use CDN services to load webfonts to your page. That will make your website load faster

Recommended services

  • Google Fonts
  • Open Font Library
  • Adobe Typekit

HEX instead of color name

You should use hex color instead of color name to ensure that the color will be rendered properly in every browser

color: #fff;

/* Bad example */
color: white;
